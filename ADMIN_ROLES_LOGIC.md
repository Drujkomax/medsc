# Логика ролей администратора

## Обновленная структура ролей и прав доступа

### Роли в системе:

1. **Director (Директор)** - полный доступ ко всем функциям:
   - CRM (лиды, сделки, задачи, канбан)
   - Продукты и категории
   - Услуги
   - Контакты
   - Управление пользователями
   - Управление сотрудниками
   - Аналитика
   - Логи активности

2. **Sales Manager (Менеджер продаж)** - фокус на продажах:
   - CRM (лиды, сделки, задачи, канбан)
   - Контакты
   - Логи активности
   - Назначение лидов

3. **Admin (Администратор)** - системное администрирование:
   - Продукты и категории
   - Услуги
   - Контакты
   - Управление пользователями
   - Аналитика
   - Логи активности
   - **НЕ имеет доступа к CRM (лиды, сделки, задачи)**

4. **Salesperson (Продавец)** - работа с назначенными лидами:
   - Только назначенные лиды
   - Базовое управление лидами

5. **User (Пользователь)** - базовый доступ

## Изменения:

- Убран доступ к лидам у роли "admin"
- Добавлена защита маршрутов с проверкой прав доступа
- CRM функции теперь доступны только Director и Sales Manager
- Admin фокусируется на системном администрировании
- Все маршруты защищены соответствующими правами доступа

## Компоненты защиты:

- `ProtectedRouteAdmin` - компонент для защиты маршрутов по правам доступа
- Обновленная навигация в `AdminSidebar` с условным отображением пунктов меню
- Защищенные маршруты в `AdminWrapper` с fallback сообщениями
